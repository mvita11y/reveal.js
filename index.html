Issue #1

 Text that does not meet the large-scale requirement has a color contrast ratio lower than 4.5:1.

Large-scale text must have a contrast ratio of at least 3:1

Large-scale text is 14 pt and bold or 18 pt

Replicate Issue

  1. Launch Chrome
  2. Visit https://www.regions.com/small-business
  3. Inspect Page
  4. Run Accessibility Audit within Lighthouse (axe-core)
  5. Identify whether failed text meets large-scale requirement by inspecting font size.

 

Low Impact, Low Priority

Low user impact, however, color contrast is a highly visible WCAG failure (easily detectable with automated checkers and by general public with lack of a11y knowledge)

Possible Solutions

  • Increase color contrast to a minimum of 4.5:1 for all text. (Large scale exemptions pose difficulties as noted above)
  • Add text-shadow to text that does not pass.

Issue #3

Responsive hamburger menu button is missing label and it's state/role is not revealed to AT users

Replicate Issue

  1. Increase browser magnification until the pages reflows (200%)
  2. Inspect element (hamburger) in the DOM
  3. Reveal the accessibility tree

High Impact, High Priority

Without a programmatic label, AT users encounter the element as an unnamed link. It's purpose, role (button) and expanded/collapsed states are not understood by AT users. 

Solution

Provide a label and follow the ARIA role, state, and properties as outlined in the navigation menu button pattern.

https://www.w3.org/TR/wai-aria-practices-1.1/#menubutton

Issue #4

Keyboard-only users are unable to close out of the hamburger menu at 200% magnification without reversing navigation (Shift + TAB)

Replicate Issue

  1. Increase browser magnification until the page reflows (200%)
  2. TAB through hamburger menu
  3. Attempt to close menu with Esc key

High Impact, High Priority

Excessive keyboard navigation can cause pain or fatigue. Two-key stroke of SHIFT + TAB can be particularly difficult. 

Solution

Menu closes with ESC key.

Issue #5

At 200% magnification, assistive technology users are not alerted of the role and state of the expandable links.

Replicate Issue

  • Increase browser magnification until the pages reflows (200%)
  • Inspect element (hamburger) in the DOM
  • Reveal the accessibility tree

High Impact, High Priority

Screen reader users simply announce the elements as links. They are not alerted of the role or state of the element.

Possible Solutions

Details Element

The HTML Details Element (<details>) creates a disclosure widget in which information is visible only when the widget is toggled into an "open" state. A summary or label can be provided using the <summary> element.

Banking
  • Business Checking
  • Business Savings
  • Business CDs
  • Business CheckCard
  • Retirement
  • Online Business Services
  • Priority Banking

ARIA Pattern for Disclosure

Keyboard Interaction

  • When the disclosure control has focus:

    • Enter: activates the disclosure control and toggles the visibility of the disclosure content.
    • Space: activates the disclosure control and toggles the visibility of the disclosure content.

WAI-ARIA Requirements

  • The element that shows and hides the content has role button.

  • When the content is visible, the element with role button has aria-expanded set to true. When the content area is hidden, it is set to false.

  • Optionally, the element with role button has a value specified for aria-controls that refers to the element that contains all the content that is shown or hidden.

https://www.w3.org/TR/wai-aria-practices-1.1/#disclosure

Issue #6

The symbols indicating that links are expandable do not have sufficient color contrast.

Replicate Issue

Use a color contrast tool to reveal the color contrast failure.

Moderate Impact,  Low Priority

Unlike text, users are unable to change the color of background images. Visual users may be unaware that the elements have expanded/collapsed sates and may visually perceive the elements simply as links. 

Solution

Darken the background color of the pseudo element.

regions-accordion-title:after, .regions-accordion-title:before {
    content: "";
    background: #131212;
    position: absolute;
    top: 50%;
